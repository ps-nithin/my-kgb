Introduction

In Finacle 7, all types of NEFT/RTGS transactions (including Cash) were carried out using HRTGS Menu. In Finacle 10, separate menus need to be used for carrying out Cash NEFT and NEFT/RTGS Transfer Transactions. The following are the Menus for initiating NEFT/RTGS.

NEFT/RTGS Menus (Detailed workflow provided below)

HPORDMThis Menu is used for initiating Customer Induced Transfer Transaction and Inter Bank Transfer Transaction
HPOOTCFor initiating Cash NEFT Transaction
HRTGSFor Inquiring transactions done in Finacle 7

NEFT/RTGS Transfer Transaction (HPORDM)

Invoke the Menu HPORDM and follow the below procedure for doing Customer induced Transfer Transaction
FunctionAdd
Payment ProductCustomer Transfer
Click on Go Button
Payment Order >> Remittance Details
Debit A/c.Enter the Operative Account from which the transaction needs to be initiated
Instrument TypeSelect CHQ - Cheque
Instrument Alpha/Serial NoEnter the Cheque Number, if applicable
Remittance CCY/Amt.Enter the Transaction Amount
Lien FlagWill be auto populated as Y
RemarksEnter Remarks, if any
Payment Order >> Beneficiary Customer
Address TypeSelect A - Name and Address
A/c. IDEnter the Beneficiary Account Number
NameEnter the Beneficiary Name
Address Line 1Enter the Address
Address Line 2Enter the Address
Address Line 3Enter the Address
Payment Order >> Account with Institution
Address TypeSelect B - BIC (IFSC Code)
BICEnter the IFSC of the Beneficiary Bank and press Tab
Payment Order >> Credit Details
Method of Payment DerivationWill be auto populated as System
Method of PaymentSelect NEFT or RTGS as applicable
After selecting Method of Payment, Click on Fetch Button
Payment Order >> Charges Details
Charge OptionWill be auto populated as Our
Charge CodeWill be auto populated according to Method of Payment selected
Net ChargesWill be auto populated as No
Repair ChargesWill be auto populated as No
Waive ChargesWill be auto populated as No/Yes based on the Debit Account Scheme
Click on View Charges Button to view the applicable charges including GST
Click Submit
Payment Order >> Remittance Info >> Expand Button
Mobile NumberEnter the Sender’s Mobile Number
Email IDEnter Email Id, If available
Click on Validate Button
Reimbursment Tab
A window will pop up for Double checking the Beneficiary Account Number and Transaction Amount
Beneficiary Credit A/c IDEnter the Beneficiary Account Number and Click Ok
Remittance CCY/Amt.Enter the Transaction Amount and Click Ok
A pop up message will be shown to visit the Payment Order Tab for Final Submission Click Ok
Ensure correctness of the Ordering Party details
Payment Order Tab
Click on Submit and Note down the Payment Order ID
