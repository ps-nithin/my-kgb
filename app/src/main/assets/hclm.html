Introduction

In Finacle 7, we are using the core security menu (SRM) to create Term Deposit, Stock and Book Debt securities. For all other securities we are using customized menus like SRMCROP, SRMMA etc. EM security is stored in the CERSAI module. In Finacle 10, core menus used to store securities have undergone changes. Following are the menus used to store security of term deposits, stock statements and book debt. There is no change in other menus.
Core Menus (Detailed workflow provided below)
HCLMFor creating Collateral ID
HSCLMFor linking/Unlinking Collateral ID to an account
HLACMFor Term Deposit Collateral Linking
HCLLCollateral Lookup - For doing Collateral Inquiry
Customized Menus (No Change)
SRMCROPFor linking Crop as Security
SRMGLFor linking Gold as Security
SRMVEHFor linking Vehicle as Security
SRMANIFor linking Animal as Security
SRMMAFor linking Vehicle as Security
SRMPBFor linking Poultry and Bee as Security
SRMINSFor linking Insurance as Security
SRMNSCFor linking NSC Certificate as Security
CERSAIPMFor creating EM Security
CERSAILMFor linking EM Security

Collateral ID Creation - Stock/Book Debt (HCLM)

This Menu is used for creating Collateral ID of Inventory (Stock), Book Debts and Term Deposits. All other security types need to be entered through corresponding security menus.
Invoke the Menu HCLM and follow the below procedure
FunctionL - Lodge
TypeSelect the applicable collateral type
B - Book Debt
T - Inventory for Stock
Click Go Button
General Tab
Collateral CodeSelect the collateral code from search
Ceiling Limit Per LinkageEnter the collateral Value
Click on Validate button
Net Value Tab
Gross ValueEnter the collateral Value
Click on Validate button
Particulars Tab
Due DateEnter the Due Date of the STOCK/BOOK DEBT
Frequency for StatementSelect the applicable Frequency of the STOCK/BOOK DEBT
Click on Submit and note down the Collateral ID generated
Before verification, Modification of the Collateral ID can be done by the created user only by invoking the Menu HCLM.
Verification of the Collateral ID can be done by any user (other than creator) having verification rights through HCLM Menu
